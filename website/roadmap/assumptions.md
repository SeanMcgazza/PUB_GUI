# ChairTime Assumptions

## Market Assumptions

### A1: Commission fatigue is real
- **Assumption:** Stylists resent commission-based booking platforms and will switch to avoid them.
- **Evidence needed:** Churn data from competitors, direct user feedback
- **Risk if wrong:** "Free" isn't enough to drive switching if existing platforms work fine
- **Validation:** Ask every signup why they chose ChairTime; track "switched from X" data

### A2: Independent stylists are underserved
- **Assumption:** Solo/freelance stylists need simpler tools than what's available.
- **Evidence needed:** User interviews, feature usage data
- **Risk if wrong:** Oversimplified for actual needs; users outgrow quickly
- **Validation:** Track feature requests, monitor user retention by business size

### A3: Irish market has room for a local player
- **Assumption:** Being explicitly Irish (language, support, understanding) is an advantage over global platforms.
- **Evidence needed:** Signup geography, user feedback on "local" value
- **Risk if wrong:** Irish users don't care; global brand recognition wins
- **Validation:** Compare conversion rates: Irish messaging vs generic messaging

## User Assumptions

### U1: Primary user is the stylist themselves
- **Assumption:** The person doing hair/beauty is also the one choosing and using booking software.
- **Impact on design:** Mobile-first, simple UX, no "admin" complexity
- **Risk if wrong:** Salon managers make decisions; need different feature set
- **Validation:** Job title/role data from signups, user interviews

### U2: Mobile is primary device
- **Assumption:** Stylists manage their business from phones, not computers.
- **Impact on design:** Responsive-first, touch-optimised
- **Risk if wrong:** Desktop users frustrated
- **Validation:** Device analytics

### U3: Instagram is primary discovery channel
- **Assumption:** Stylists share booking links on Instagram; clients find them there.
- **Impact on design:** Easy link copying, Instagram-friendly booking pages
- **Risk if wrong:** Need to invest in other marketing channels
- **Validation:** Referral source tracking

## Product Assumptions

### P1: Free tier is sustainable
- **Assumption:** Conversion to paid (SMS, teams) will fund free tier costs.
- **Risk if wrong:** Unsustainable business model; need to add limits or raise prices
- **Validation:** Unit economics analysis, conversion rate tracking

### P2: Email reminders are sufficient for free tier
- **Assumption:** Free users accept email reminders; SMS is a reasonable upsell.
- **Risk if wrong:** Email isn't opened; no-shows remain high; product seems ineffective
- **Validation:** Reminder open rates, no-show rates by plan type

### P3: Clients don't need accounts
- **Assumption:** Clients booking don't want to create accounts or download apps.
- **Risk if wrong:** Friction in returning to rebook; client retention suffers
- **Validation:** Repeat booking rates, user feedback

### P4: €12/month is acceptable for premium
- **Assumption:** Price point is affordable for solo stylists, competitive with alternatives.
- **Risk if wrong:** Too high = no conversions; too low = not sustainable
- **Validation:** Conversion rates, churn rates, competitive analysis

---

## Copy/Marketing Assumptions

### M1: Zero commission is the killer message
- **Assumption:** Leading with "no commission" resonates most strongly.
- **Risk if wrong:** Other messages (time saved, professionalism) might convert better
- **Validation:** A/B test headlines; survey new users on primary motivation

### M2: Anti-platform positioning works
- **Assumption:** "We're not like Fresha" resonates without seeming petty.
- **Risk if wrong:** Comparisons might seem desperate or confuse users who don't know competitors
- **Validation:** User feedback, A/B test comparison content

### M3: Word-of-mouth is primary growth
- **Assumption:** Stylists recommend tools to each other; satisfied users drive signups.
- **Risk if wrong:** Need paid acquisition or other channels
- **Validation:** Referral tracking, "how did you hear about us" data

---

## Technical Assumptions

### T1: Web app is sufficient
- **Assumption:** PWA/responsive web meets user needs; native apps aren't necessary.
- **Risk if wrong:** Users expect native apps; adoption suffers
- **Validation:** Direct user feedback, app store search volume for competitors

### T2: Google Calendar sync covers most needs
- **Assumption:** GCal is the calendar most stylists use.
- **Risk if wrong:** Apple Calendar users, Outlook users underserved
- **Validation:** Feature requests, calendar sync usage rates
